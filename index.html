this.web 《甚麼是 API》
在剛學前端時，對 API 的了解很模糊
只知道用 fetch 來獲取特定網址的資料就是串 API，但 API 到底是什麼卻不清楚

其實 API 就像一個提供特定服務的窗口
像前端在取得後端的資料時
就要經過一個網址來取得
這個網址就是一個 API
所以 API 就是兩個應用程式之間進行通訊的橋樑
而且它會定義應用程式之間互相操作的規則和約定

當然 API 也不只是一個網址，也可以是一個函數或方法
像是 setTimeout 就是瀏覽器提供給 JS 的一個 API
讓開發人員可以使用計時的功能

其它內容就放在貼文中了！
那這次就這樣，下次見～！

kayshih.dev 一定常聽到想找海外工作那一定得學React，我從React 15版就開始寫也大概寫六年了，這篇是想聊聊我對React為什麼能紅這麼久以及會繼續紅多久的一些看法

根據November 2022 Hacker News Hiring Trends 數據圖，因為2023的圖表還沒出來，所以先看2022/11的，看圖就可以發現到了2022，React職缺數還是甩其他框架好幾條街。

真的沒其他框架比它好用嗎？
到底有什麼原因能讓React紅這麼久？

◾️無痛升級
React無痛升級絕對是三個主要前端框架裡面做得最優秀的，幾乎是完全不用改什麼大架構都能做到升級，以改動最大的15 -> 16版本來說，React也提早與開發者告知某些API即將被棄用，既使到了Hook出現也不影響舊有的寫法。

◾️生態系完整
對於大部分的開發者而言都是在幫公司打造產品，做產品的過程我們都希望能少重新造輪子，如果有既有的Library當然是先拿過來用，而圍繞在React生態系的第三方Library真的非常完整，幾乎你要做任何UI或是功能都找得到現成的套件。

◾️Vercel
Vercel團隊一直都與React核心團隊有密切合作，React核心團隊的成員也有跳過去Vercel團隊以負責未來React的發展方向，2021年12月vercel就有發布文章提到未來會全力支援React的發展，React能持續這麼夯，很大部分要感謝Vercel團隊以及NextJS的出現，Nextjs真的讓React開發者體驗變得很優秀，幾乎什麼專案都能打。

除了Vercel團隊以外，由Kent C. Dodds跟其他幾位大神開發的Remix也是現在很紅的React框架，順帶一提Nextjs & Remix conference都是我目前必追的React Conference

React Team常常在社群上被嫌的就是有點太過於走火入魔，把框架搞得overengineering還有像是useEffect被嫌的一塌糊塗，但我相信React team自己也知道這些問題，不過要有些走火入魔的人走在前頭才有辦法把這個領域玩的更深，不管是接下來的Server Component或是React Forget，我相信React還會有更多有趣的東西出，反正最終受惠的都是我們這些平凡的開發者。

你說React還能唱秋多久？

front_end_divers #fedv_js_8
👩🏻‍💻這禮拜不小心拖了一下稿 (其實是因為在研究新玩具錯過發文時間，再找時間跟大家分享😂)今天來跟大家介紹大家一定都遇過的~~變數宣告！➙來比較一下 var, let, 跟 const 三者的差異：

1️⃣ 提升 ( hoist )
因為 var 會變數提升( hoisting ) ，在一開始就會被提升到 scope 的一開始，先於程式碼的執行，來進行變數的宣告，並賦予其 undefined 為 default value。所以儘管是先執行到 console.log 也可以將 friend1 印出 undefined。

而 let 跟 const 其實也會變數提升，但是不同的地方在於將使用 let 及 const 宣告的變數提升後，這些變數並不會被賦予 default value，因此如果在真正執行到這些變數被賦值之前去使用它們，會收到他們還沒被 初始化( innitialize ) 的 error。 因此直到他們被賦值的那一行 code 被執行到，這些變數才可開始被使用。這樣的特性也叫做 TDZ ( Temporal Dead Zone)。
p.s. JavaScript 的 hoist 與上一篇介紹的npm hoisting 不同，想了解可以參考
#fe_divers_tools_1 ）

2️⃣ 全域物件 ( global object )
在最外層使用 var 來進行變數宣告時，會自動在全域物件中建立一個 property，所以可以在 global 用 this. 的方式來取得變數的 value。而 let 以及 const 則不會有這樣的機制。

3️⃣ scope 規則
🔹var : global scope 或 function scope，容易污染到環境
🔹let & const : block scope ，在 if, for loop, while loop 宣告的變數都會被關在裡面

4️⃣ redeclare and reassign
🔹用 var 宣告的變數：可以重複宣告，可以重新賦值
🔹用 let 宣告的變數：不能重複宣告，可以重新賦值
🔹用 const 宣告的變數：不能重複宣告，不能重新賦值

所以 let 和 const 的出現可以避免 var 不必要的 hoist，解決使用 var 時容易污染環境變數的缺點(很多小 bug 都是在這裡發生的 !)
